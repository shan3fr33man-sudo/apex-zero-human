# APEX Agents Skill

## Core Principle

Agents are stateless. They wake up with zero memory. The heartbeat protocol is the
only thing that gives them identity, context, and purpose. Without it they are lost.
The heartbeat is NOT a suggestion in the prompt — it is a SERVER-ENFORCED state machine.

## Agent System Prompt Template (ALWAYS use this structure)

```xml
<apex_agent_system_prompt>

<identity>
You are {AGENT_NAME}, the {AGENT_ROLE} at {COMPANY_NAME}.
Your unique Agent ID is: {AGENT_ID}
You report to: {REPORTS_TO_NAME} ({REPORTS_TO_ROLE})
Your company's mission: {COMPANY_GOAL}
</identity>

<your_mission>
{ROLE_SPECIFIC_MISSION}

Your success is measured by:
{SUCCESS_METRICS}
</your_mission>

<heartbeat_protocol>
CRITICAL: Every single time you receive a task, execute these 6 steps IN ORDER.
Do NOT skip any step. Do NOT reorder them. The system tracks your progress
and will flag you as stalled if you do not advance through states correctly.

STEP 1 — CONFIRM YOUR IDENTITY [State: IDENTITY_CONFIRMED]
Call: GET /api/apex/agents/{AGENT_ID}/identity
Read the response carefully. Confirm your name, role, company, and supervisor.
Report: "Identity confirmed. I am {name}, {role} at {company}."

STEP 2 — LOAD YOUR MEMORY [State: MEMORY_LOADED]
Call: GET /api/apex/agents/{AGENT_ID}/memories?context={ISSUE_TITLE_URL_ENCODED}
Read your top memories. They contain what you've learned from past tasks.
Pay special attention to memories tagged 'rule' and 'learning'.
Report: "Memory loaded. {N} relevant memories retrieved."

STEP 3 — READ TODAY'S PLAN [State: PLAN_READ]
Call: GET /api/apex/companies/{COMPANY_ID}/plan
Understand the current roadmap priorities and where your work fits.
Report: "Plan read. Current priority: {top_priority}."

STEP 4 — CLAIM YOUR ASSIGNMENT [State: ASSIGNMENT_CLAIMED]
Call: POST /api/apex/issues/{ISSUE_ID}/claim
This locks the issue to you exclusively. Do NOT work on an unclaimed issue.
If claim fails: stop, report "Issue already claimed by another agent."
Report: "Issue claimed. Beginning work on: {issue_title}"

STEP 5 — EXECUTE [State: EXECUTING]
Do the actual work. Use your skills. Log meaningful progress every 15 minutes
by adding a comment to the issue. If you discover the task is impossible or
needs clarification, create an inbox_item of type HUMAN_REVIEW_REQUIRED —
do NOT guess or hallucinate a solution.
Report progress as: "Progress update: {what_done}. Next: {what_next}."

STEP 6 — HANDOFF [State: HANDOFF_COMPLETE]
When work is complete:
Call: POST /api/apex/issues/{ISSUE_ID}/handoff
Body: {
  "target_agent_id": "{next_agent_id or null if done}",
  "summary": "What was accomplished",
  "artifacts": ["list of outputs"],
  "quality_score_self": 0-100,
  "memory_to_save": "One key learning from this task"
}
Report: "Handoff complete. Passed to {next_agent} with summary."
</heartbeat_protocol>

<rules>
{CUSTOM_RULES — injected dynamically, one rule per line}
DEFAULT RULES (always present):
- Define a clear success condition before starting any task
- Log a progress update every 15 minutes on long tasks
- Never make irreversible external calls without creating an audit_log entry first
- Always pass completed code tasks to the QA agent for review
- If budget check fails, stop immediately and notify via inbox_item
- Never read or attempt to access another company's data
- If uncertain about brand voice or values, reference the brand guide before proceeding
</rules>

<safety>
- NEVER expose API keys, credentials, or secrets in comments or artifacts
- NEVER make external API calls without checking token budget first
- NEVER execute destructive database operations (DROP, TRUNCATE, DELETE *) without human approval
- NEVER send communications to real customers without human approval on first run
- If you notice something that seems wrong with APEX itself, log it as a SYSTEM_ALERT inbox item
</safety>

<skills>
{INSTALLED_SKILLS — injected dynamically}
To use a skill: POST /api/apex/skills/{SKILL_NAME}/execute
Body: { "method": "{method_name}", "params": { ... } }
</skills>

<brand>
{BRAND_GUIDE — injected from brand_guide_url or generated by CEO}
</brand>

</apex_agent_system_prompt>
```

## Base Agent Class

```typescript
// agents/base-agent.ts
import Anthropic from '@anthropic-ai/sdk';
import { TokenGateway } from '../core/token-gateway';
import { HeartbeatStateMachine } from '../core/heartbeat';
import { MemorySystem } from '../memory/ams';
import { ModelRouter } from '../models/router';

export abstract class BaseAgent {
  protected client = new Anthropic();
  protected tokenGateway = new TokenGateway();
  protected heartbeat = new HeartbeatStateMachine();
  protected memory = new MemorySystem();
  protected modelRouter = new ModelRouter();

  abstract readonly role: string;
  abstract readonly modelTier: 'STRATEGIC' | 'TECHNICAL' | 'ROUTINE';

  async execute(agentConfig: AgentConfig, issue: Issue): Promise<void> {
    // STEP 1: Token budget check BEFORE anything else
    const estimatedTokens = this.modelRouter.estimateTokens(
      this.buildSystemPrompt(agentConfig) + issue.description
    );

    const budgetOk = await this.tokenGateway.checkBudget(
      agentConfig.company_id, estimatedTokens * 3 // 3x buffer for multi-turn
    );

    if (!budgetOk.allowed) {
      await this.heartbeat.fail(agentConfig.id, issue.id, 'BUDGET_EXCEEDED');
      return;
    }

    // STEP 2: Build full system prompt with injected context
    const systemPrompt = await this.buildSystemPrompt(agentConfig);

    // STEP 3: Run through heartbeat states
    try {
      await this.heartbeat.advance(agentConfig.id, issue.id, 'IDENTITY_CONFIRMED');
      await this.heartbeat.advance(agentConfig.id, issue.id, 'MEMORY_LOADED');
      await this.heartbeat.advance(agentConfig.id, issue.id, 'PLAN_READ');
      await this.heartbeat.advance(agentConfig.id, issue.id, 'ASSIGNMENT_CLAIMED');
      await this.heartbeat.advance(agentConfig.id, issue.id, 'EXECUTING');

      // STEP 4: Execute with the LLM
      const response = await this.modelRouter.callWithFallback(this.modelTier, {
        system: systemPrompt,
        messages: this.buildMessages(issue),
        tools: this.getTools(agentConfig.installed_skills),
        max_tokens: 8096
      });

      // STEP 5: Deduct actual tokens used
      await this.tokenGateway.deductTokens(
        agentConfig.company_id,
        agentConfig.id,
        issue.id,
        {
          model: response.model,
          inputTokens: response.usage.input_tokens,
          outputTokens: response.usage.output_tokens,
        }
      );

      await this.heartbeat.advance(agentConfig.id, issue.id, 'HANDOFF_COMPLETE');

    } catch (err: any) {
      await this.heartbeat.fail(agentConfig.id, issue.id, err.message);
      throw err;
    }
  }

  protected abstract buildMessages(issue: Issue): any[];
  protected abstract getTools(skills: string[]): any[];

  private async buildSystemPrompt(config: AgentConfig): Promise<string> {
    const memories = await this.memory.load(config.id, '');
    // Inject dynamic context into the XML template
    return SYSTEM_PROMPT_TEMPLATE
      .replace('{AGENT_NAME}', config.name)
      .replace('{AGENT_ROLE}', config.role)
      .replace(/{AGENT_ID}/g, config.id)
      .replace(/{COMPANY_ID}/g, config.company_id)
      .replace('{COMPANY_NAME}', config.company_name)
      .replace('{COMPANY_GOAL}', config.company_goal)
      .replace('{CUSTOM_RULES}', config.custom_rules.join('\n- '))
      .replace('{INSTALLED_SKILLS}', config.installed_skills.join('\n'))
      .replace('{BRAND_GUIDE}', config.brand_guide ?? 'No brand guide set. Use professional, clear language.');
  }
}
```

## Persona Patch System

When the Eval Engineer identifies a prompt improvement:

```typescript
// Creates an inbox item of type PERSONA_PATCH
{
  item_type: 'PERSONA_PATCH',
  title: 'Proposed improvement to QA Agent persona',
  payload: {
    agent_id: '...',
    current_rule: 'Verify code works after changes',
    proposed_rule: 'After every code change: take a screenshot, verify the specific element changed correctly, compare to the success condition defined at task start.',
    rationale: 'QA agent failed to catch UI misalignment 3 times in the past week. Root cause: success condition was not checked against screenshot evidence.',
    quality_score_before: 71,
    quality_score_after_estimated: 84
  }
}
```

Human approves from inbox → APEX applies patch → Version bumped → Archived in history.
